# Supported EC2 instance types using Terraform

## Backend EC2 instances

The EC2 instance models can operate as **backend**, **client,** or **converged** instances. The default EC2 instance model for backends is **i3en.2xlarge**.

<table><thead><tr><th width="237">EC2 instance type</th><th>Supported instances</th></tr></thead><tbody><tr><td>I3en</td><td>i3en.2xlarge, i3en.3xlarge, i3en.6xlarge, i3en.12xlarge, i3en.24xlarge</td></tr></tbody></table>

## Client EC2 instances

The following EC2 instance models can operate as **client** instances. The default EC2 instance model for clients is **c5.2xlarge**.

{% hint style="info" %}
* Support for WEKA client over UDP mode is extended to any Intel or AMD CPU-based instance type, provided that the instance type meets the resource requirements specified in the [prerequisites-and-compatibility.md](../../prerequisites-and-compatibility.md "mention") topic.
* Any backend instance can also be a client instance.
{% endhint %}
